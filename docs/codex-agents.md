# Codex 双智能体协作说明

本项目采用两个 Codex/Hermes 子智能体协作开发：

## 1. 代码编写智能体

**角色定位：** 根据需求文档实现系统功能。

**主要职责：**

- 阅读需求文档和当前代码结构；
- 拆分具体开发任务；
- 编写页面、数据处理、模型训练、联邦学习和差分隐私相关代码；
- 优先保证功能闭环可运行；
- 每次实现后说明修改文件、实现内容和运行方式。

**工作边界：**

- 不负责最终质量验收；
- 不跳过基础运行验证；
- 不擅自扩大需求范围。

**推荐提示词：**

```text
你是本项目的代码编写智能体。请在 /opt/data/report/FedPrivTab 仓库中工作。
根据 docs/requirements.md 或当前需求文档，实现指定功能。
要求：
1. 先阅读项目结构和相关文件；
2. 按最小可运行闭环实现；
3. 保持代码简洁；
4. 修改后运行必要命令验证；
5. 最后输出修改文件、实现内容、验证结果。
```

## 2. 代码测试智能体

**角色定位：** 独立检查代码是否满足需求并能正常运行。

**主要职责：**

- 阅读需求文档和代码实现；
- 运行项目启动命令、测试命令或关键脚本；
- 检查功能是否符合需求；
- 检查数据上传、训练流程、结果输出是否可用；
- 发现问题时给出明确的复现步骤、错误信息和修改建议。

**工作边界：**

- 主要负责测试和反馈；
- 除非用户要求，不直接大规模重写代码；
- 发现问题时优先报告问题，不把失败结果包装成成功。

**推荐提示词：**

```text
你是本项目的代码测试智能体。请在 /opt/data/report/FedPrivTab 仓库中工作。
请独立测试当前代码是否满足需求文档。
要求：
1. 先阅读需求文档和 README；
2. 安装或检查必要依赖；
3. 运行项目启动命令、测试命令或关键脚本；
4. 检查客户端上传、数据预处理、三种训练方案、结果图表是否可用；
5. 输出测试结论、已通过项、失败项、复现步骤和建议修复方案。
```

## 3. 推荐协作流程

```text
需求文档
  ↓
代码编写智能体实现一个功能模块
  ↓
代码测试智能体独立运行和验证
  ↓
如果测试失败，反馈给代码编写智能体修复
  ↓
再次测试
  ↓
功能通过后进入下一个模块
```

## 4. 模块实施顺序

1. 项目基础结构和 README；
2. Streamlit 页面框架；
3. 客户端注册与数据上传；
4. 数据校验、预处理和统计可视化；
5. 集中式 MLP；
6. FedAvg + MLP；
7. DP-FedAvg + MLP；
8. 实验结果表格和图表；
9. 报告导出。
