# FedPrivTab 当前实现需求文档

## 1. 项目概述

FedPrivTab 是一个面向 Non-IID 表格数据场景的联邦学习与差分隐私实验系统。系统后端基于 Flask 提供认证、数据预处理、训练和报告接口；前端使用 Vue 单页应用提供登录、数据分析、数据预处理、实验训练和结果分析等页面。

系统围绕三类训练方案进行对比：

| 方案 | 说明 | 用途 |
|---|---|---|
| 集中式 MLP | 将训练数据集中后统一训练 | 作为性能参考上限 |
| FedAvg + MLP | 多客户端本地训练，服务端按样本量加权聚合 | 观察普通联邦学习效果 |
| DP-FedAvg + MLP | 在 FedAvg 基础上对客户端更新裁剪并加噪 | 分析隐私保护与性能损失 |

## 2. 当前系统架构

| 层级 | 当前实现 |
|---|---|
| 前端 | Vue 3 + Vite 单页应用，部署在 8501 端口 |
| 后端 | Flask API，部署在 5000 端口 |
| 认证 | SQLite 本地认证库，支持 session_id |
| 预处理 | Flask `/preprocess` 后端接口，支持 multipart CSV 文件和 JSON records |
| 训练 | Flask `/train` 调用 PyTorch/Scikit-learn 训练逻辑 |
| 报告 | Flask `/report` 输出 Markdown 实验报告 |
| 部署 | systemd 管理 `fedprivtab-api` 与 `fedprivtab-ui` 服务 |

说明：`fedprivtab-ui` 服务承载 Vue 构建产物。

## 4. 页面需求

### 4.1 登录页

- 展示“登录工作台”。
- 支持用户名、密码输入。
- 登录成功后根据角色进入首页。
- 登录失败展示明确错误提示。

### 4.2 首页

- 展示实验概览。
- 展示客户端数量和训练结果数量。
- 展示三类方案：集中式 MLP、FedAvg + MLP、DP-FedAvg + MLP。
- 不展示冗余“数据状态”指标。

### 4.3 客户端管理页

- 仅管理员和实验研究人员可见。
- 系统固定 4 个客户端账号：`client-1` 至 `client-4`。
- 不支持新增客户端。
- 不支持删除客户端。
- 顶部指标只保留“客户端账号”数量。
- 客户端账号清单内提供修改密码能力。
- 不展示“修改我的密码”入口。

### 4.4 数据分析页

- 作为独立功能页面，不依赖数据预处理页。
- 支持上传指定 CSV 文件进行分析。
- 上传文件时展示 loading/刷新状态。
- 上传后保留当前分析文件状态。
- 展示样本数、字段数、数值特征数、缺失值数量。
- 展示完整统计摘要。
- 字段分布图支持下拉选择任意字段。
- 单特征分布支持下拉选择任意字段。
- 特征均值展示全部适用数值特征。
- 相关性热力图展示全部适用数值特征；非数值字段需说明未纳入。
- 大数据量图表渲染时可采样展示，统计摘要仍基于完整数据。

### 4.5 数据预处理页

- 支持上传 CSV 文件。
- 上传后展示 loading/刷新状态，并刷新目标变量、缺失值摘要、数值标准化和数据预览。
- 上传文件状态在页面切换后保留。
- 不提供“生成示例数据”入口。
- 支持选择目标变量。
- 缺失值处理按列配置，支持：删除含缺失行、均值填充、中位数填充、众数填充。
- 缺失值处理方式默认使用系统推荐策略：
  - 缺失率较高默认推荐删除；
  - 数值或数字字符串字段默认推荐中位数；
  - 类别字段默认推荐众数。
- 数值标准化按列配置，支持不处理、StandardScaler、MinMaxScaler。
- 数值或数字字符串字段应能被识别为数值型候选字段。
- 一键处理数据时展示“处理中”状态。
- 一键处理优先通过后端 `/preprocess` multipart CSV 文件接口执行，避免前端按行 JSON 传输导致卡顿。
- 处理成功后保存处理版本，并刷新“处理版本记录”。
- 处理失败时展示明确错误信息。

### 4.6 实验训练页

- 仅管理员和实验研究人员可见。
- 支持实验参数配置：客户端数量、通信轮数、集中式 Epoch、隐藏层数量、隐藏层神经元、激活函数、学习率。
- 支持高级联邦与差分隐私参数：Batch Size、本地 Epoch、客户端采样比例、Dirichlet alpha、聚合方式、裁剪阈值、噪声倍率、epsilon、delta。
- 支持勾选训练方案：集中式 MLP、FedAvg + MLP、DP-FedAvg + MLP。
- 支持选择数据版本。
- 点击“开始训练”后调用后端 `/train` 接口并展示训练结果。

### 4.7 结果分析页

- 展示各训练方案结果。
- 展示 Accuracy、Precision、Recall、F1、AUC 等指标。
- 展示训练曲线摘要、混淆矩阵、客户端分布和差分隐私参数。
- 支持调用 `/report` 生成 Markdown 报告。

## 5. 后端 API 需求

| 接口 | 方法 | 说明 |
|---|---|---|
| `/health` | GET | 健康检查 |
| `/auth/login` | POST | 登录认证 |
| `/auth/logout` | POST | 退出登录 |
| `/auth/status` | GET | 查询登录状态 |
| `/users` | GET | 管理员/研究人员查看用户列表 |
| `/users` | POST | 兼容保留的账号创建接口 |
| `/users/<username>/status` | PATCH | 兼容保留的账号启用/禁用接口 |
| `/sample-data` | GET | 生成示例数据，用于 API 测试和训练兜底 |
| `/validate` | POST | 校验表格数据 |
| `/preprocess` | POST | 后端数据预处理，支持 JSON records 和 multipart CSV 文件 |
| `/train` | POST | 执行集中式/FedAvg/DP-FedAvg 训练 |
| `/report` | POST | 根据训练结果生成 Markdown 报告 |

## 6. 部署需求

- Flask API 运行在 `5000` 端口。
- Vue 前端运行在 `8501` 端口。
- 前端构建产物由静态 HTTP 服务托管。
- 前端上传流程必须由 Vue 静态页面和 Flask API 完成，不依赖外部 telemetry/webhook 请求。
- 远端服务用户为当前部署环境的 `ubuntu`。
- 部署后必须验证：
  - `http://110.40.166.66:5000/health` 返回 `status=ok`；
  - `http://110.40.166.66:8501` 页面可访问；
  - `/preprocess` multipart CSV 调用成功。

## 7. 验收标准

- 单元测试全部通过。
- UI 自动化覆盖登录、导航、权限、数据分析页、数据预处理页、实验训练页和结果分析页关键文案。
- 上传 CSV 后无第三方 webhook 请求卡顿。
- 数据预处理不再按行上传大 JSON，而是优先走 multipart 文件上传。
- 客户端用户看不到客户端管理页和实验训练页。
- 管理员/研究人员可以完成从数据分析、预处理、训练到报告导出的完整流程。
